# Breakout Filter Playbook — 假突破过滤执行手册

> **这份文档是你每次要追突破时必须打开的"副驾驶"**  
> 核心目标：把假突破踩中率从当前估计的 60%+ 降到 30% 以下  
> 基于：TRADING_SYSTEM_SPEC.md 第三章的四层过滤体系

---

## 零、一句话纲领

> **你不是"看到突破就该入场的人"，你是"看到突破会多问三个问题再决定的人"。**

三个问题是：
1. 今天适合追突破吗？（Regime）
2. 这次突破有增量资金支持吗？（Volume + 多周期）
3. 我能不能等回踩再入场？（Retest）

三个都"是" → 入场。任何一个"否" → 跳过。

## 一、五种典型假突破模式（识别清单）

### 模式 A：Opening Drive Fade（开盘冲高/砸低后反转）

**ASCII 示意**：
```
价格
 ↑         ╭─╮
 │        ╱   ╲___________  ← 冲高后快速回落
 │       ╱                   ← 假突破
 │      ╱
 │ ___ ╱
 │    ← 昨日高点
 └──────────────────────→ 时间
    9:30  9:45  10:00  10:30
```

**特征**：
- 开盘 15-45 分钟内快速突破昨日高点（或关键阻力）
- 突破 bar 往往是 1min 一根大阳线
- 之后 30 分钟内回到原阻力位下方

**为什么常见**：算法利用开盘订单簿不平衡，故意推过阻力位触发散户追入单，然后反手平仓

**你的数据证据**：29% 订单在 09:30-10:00，这个时段胜率最低

**应对**：
- **09:30-10:00 硬禁令：不追任何突破**
- 10:00 之后才能考虑（且仍需层 2、层 3 过滤）

---

### 模式 B：Low Volume Breakout（无量假突破）

**ASCII 示意**：
```
价格                           
 │        ╭──────  ← 突破了
 │       ╱
 │______╱         
 │               
 │                
成交量（条）       
 │  ▌▌▌▌▌▌▌▌▌▌    ← 过去 20 bar 常态量
 │            ▌    ← 突破 bar 的量！比常态还小
 └──────────────────→ 时间
```

**特征**：
- 价格突破阻力位
- 但突破那根 bar 的 volume < 过去 20 bar avg × 1.2

**为什么是假**：没有增量资金支持，是前一浪的惯性或个别大单扫盘导致

**识别规则**（硬性数字）：
- volume / avg_20_bar < 1.2 → **几乎必假**
- 1.2 - 1.5 → **弱确认**
- 1.5 - 2.0 → **标准确认**
- \> 2.0 → **强确认**

**你能直接照抄的 Python 代码**：
```python
def check_volume_confirmation(df_2min):
    """df_2min 是最近 21 根 2min K 线 (pandas DataFrame)。
    最后一根是当前突破 bar。"""
    if len(df_2min) < 21:
        return False, "数据不足"
    avg_20 = df_2min['volume'].iloc[-21:-1].mean()
    current = df_2min['volume'].iloc[-1]
    ratio = current / avg_20
    if ratio < 1.2:
        return False, f"无量突破 ({ratio:.2f}x)，跳过"
    if ratio < 2.0:
        return True, f"弱量确认 ({ratio:.2f}x)，仅小仓位试水"
    return True, f"强量确认 ({ratio:.2f}x)，可以入场"
```

### 模式 C：Divergence Breakout（个股逆大盘突破）

**ASCII 示意**：
```
  个股：      _____────  ← 突破
            ╱
           ╱
  ─────────

  SPY：   ─────────╲
                   ╲___  ← SPY 在跌！
                     ╲__ 
```

**特征**：
- 个股突破创新高
- 但同一时间 SPY 或板块 ETF（XLK 等）在下跌或横盘

**为什么常见**：个股内部消息或者大买单推动，缺乏大盘流动性支撑，反弹结束后会被大盘拖下来

**识别规则**：
- 个股过去 30 分钟 return - SPY 过去 30 分钟 return < 0 → **放弃**
- 差值 0 - 0.3% → **试仓小**
- 差值 > 0.3% → **正常**

**代码**：
```python
def check_rs_vs_spy(symbol, window_min=30):
    """返回个股和 SPY 过去 N 分钟的收益率差值（正值越大越强）"""
    stock_ret = get_return_last_n_min(symbol, window_min)
    spy_ret   = get_return_last_n_min('SPY', window_min)
    return stock_ret - spy_ret  # > 0 强，< 0 弱
```

### 模式 D：Range Extreme Pop（箱体顶部试探）

**ASCII 示意**：
```
价格            ←── 突破箱体顶
 │      ╭╮╭╮╭──╮ ╱
 │   ╭─╯╰╯╰╯   ╲╱       ← 快速回落
 │  ╱                        
 │ ╱   （3 天箱体）         
 │╱                          
 └────────────────────→ 时间
```

**特征**：
- 价格在一个区间（如 $180-$185）震荡了 2-5 个交易日
- 某个时刻突破区间高点 $185
- 但成交量没有明显放大

**为什么是假**：箱体震荡说明多空势均力敌，突破没有新催化剂一般不持续

**识别规则**：
- 如果最近 5 个交易日日内高点的**方差 < 1%**（典型箱体）
- 突破时 volume < 2x → **几乎必假**
- 突破伴随明确消息催化（财报、分析师上调）→ 另算

---

### 模式 E：Pre-News Breakout（事件前假突破）

**ASCII 示意**：
```
时间轴：
 14:00          15:00 FOMC 公布
 ──────╭────╮──────────
       │ 突破 │  ← 14:30 出现的"突破"几乎全假
       ╰────╯
```

**特征**：
- 在重大事件（FOMC / CPI / 财报）公布前 30-120 分钟
- 市场会出现"试探性突破"
- 无论哪个方向都不可信，因为机构已经在减仓/调整

**识别规则**：
- 今日有 FOMC 决议（14:00 ET） → **13:00-14:00 禁止追突破**
- 今日有 CPI/PPI（8:30 ET） → **开盘第一小时禁止追突破**
- 个股今晚盘后财报 → **盘后最后 2 小时禁止追突破**

## 二、层层过滤决策树（对着检查）

每次看到"突破"时，从上到下逐个检查：

```
检测到突破
    │
    ▼
[Q1] 今天 Regime Score ≥ 55？
    ├─ 否 → ❌ 跳过
    └─ 是 → 继续
        │
        ▼
[Q2] 当前时间是否在"禁止时段"？
    ├─ 09:30-10:00 → ❌ 跳过
    ├─ 11:30-13:30 午盘 → ⚠️ 仅放 retest，不追
    ├─ FOMC 日 13:00-14:00 → ❌ 跳过
    ├─ 财报日个股盘后 → ❌ 跳过
    └─ 否则 → 继续
        │
        ▼
[Q3] Volume 确认？
    ├─ < 1.2x → ❌ 跳过
    ├─ 1.2-1.5x → ⚠️ 降级为 1/3 仓
    ├─ 1.5-2.0x → ⚠️ 标准仓位但只做 retest
    └─ > 2.0x → 继续
        │
        ▼
[Q4] 多周期一致性？
    ├─ 2min+5min+15min+daily 都向上（4 个）→ ✅ 最强
    ├─ 3 个向上 → ✅ 可以
    ├─ 2 个向上 → ⚠️ 降级
    └─ < 2 个向上 → ❌ 跳过
        │
        ▼
[Q5] RS vs SPY？
    ├─ > +0.3% → ✅ 继续
    ├─ 0 - +0.3% → ⚠️ 降级
    └─ < 0 → ❌ 跳过
        │
        ▼
[Q6] 选择入场方式
    ├─ 分数最高（Q3-Q5 全强）→ 分批试仓 1/3
    ├─ 中等 → 等 Retest
    └─ 边缘 → Shadow Trade 记录，不实盘
```

## 三、Retest 入场的具体操作

### 3.1 Retest 的定义

**定义**：价格突破阻力位 P，上涨一段，回落到 P 附近（P ± 0.5%），**不跌破 P**，再次向上 → 入场。

### 3.2 具体步骤（每次追突破都按这个做）

**Step 1：识别突破位**
- 当价格突破某个关键位（昨日高点、日内高点、箱体上沿等），立即标记这个价位为 `P_break`
- 在笔记/系统中记录：`{symbol}: P_break = $X, time = 14:32`

**Step 2：不入场，等**
- 设置闹钟：30 分钟后或者价格回踩 `P_break` 时提醒
- **这是最难的一步**。95% 的散户做不到。

**Step 3：观察回踩行为**
- 如果价格回踩到 `P_break ± 0.5%` 范围：
  - ✅ **回踩不破（价格在 P_break 之上企稳）** → 进入 Step 4
  - ❌ **回踩击穿 P_break 且 5 分钟内不回来** → 假突破已证实，庆祝自己没追

**Step 4：确认反弹 bar**
- 回踩到 P_break 附近后，等待一根**带量的阳线**（volume > avg × 1.5，close > open）
- 这根 bar 的出现说明资金在这个位置承接

**Step 5：入场**
- 在确认 bar 的收盘价入场（或下一根 bar 开盘）
- 止损设在 `P_break × 0.995`（即突破位下方 0.5%）
- 期权选择：在 Retest 确认后，看当时的标的价格和你的方向判断，选合适 DTE 的合约（不要因为"已经涨了"就选太激进的 OTM）

**Step 6：如果 30 分钟没回踩**
- **放弃**。一路上涨不回踩的极强趋势，你不是唯一看到的人，追入风险极高
- 宁可错过，不要追高

### 3.3 Retest 的心理挑战

**常见心理障碍**：
- "万一它不回踩一直涨怎么办？" → 那就让它涨，你的错过不会亏损
- "它已经涨 2% 了，再等就晚了" → 历史上追高 2% 后的胜率比回踩入场低 30%
- "这是我等了一天的机会" → 情绪语言，不是数据语言

**心理对抗技巧**：
- 在墙上贴一条纸条：**"不追高。回踩或放弃。"**
- 每当手指要下单时，默念一遍
- 如果还是下了单，Journal 里标注 `entry_method = chase`，月度复盘时对比 chase vs retest 的胜率差

## 四、分批试仓法（混合方案）

当你 Q3-Q5 都通过但没法等 Retest 时，用这个折衷：

### 4.1 规则

```
突破发生 + 层 1/2 全通过：
    试仓 1/3 总仓位（假设原计划买 12 张 call，先买 4 张）
    设置止损：P_break × 0.995

如果 Retest 成功：
    加仓 2/3（再买 8 张），同一止损
    
如果 Retest 失败（击穿 P_break）：
    止损离场，只亏 1/3 仓位
```

### 4.2 预期效果

假设假突破概率 60%：
- **全仓追入**：60% × -100% 亏损 = 期望 -60%
- **分批试仓**：60% × -33% 亏损 + 40% × +150% 盈利 = 期望 +40%

数字是示意的，但**结构上**分批试仓比全仓追入在任何胜率下都更优。

## 五、Journal 字段标注规范

每笔"意图为突破"的交易，必须填以下字段：

| 字段 | 取值 | 什么时候填 |
|--|--|--|
| `trade_style` | `breakout_chase` / `breakout_retest` / `breakout_split` (分批) / `pullback_buy` / `mean_reversion` / `other` | 入场时 |
| `pre_filter_pass` | bool | 入场时（Regime Score ≥ 55 且时段合规） |
| `breakout_volume_multiple` | float | 入场时（查 2min K 线） |
| `timeframe_alignment` | 0-4 | 入场时 |
| `rs_vs_spy_30min` | float | 入场时 |
| `entry_was_retest` | bool | 入场时 |
| `was_fake_breakout` | bool | 出场后由 AI + 人工判定 |

**判定 `was_fake_breakout` 的规则**：
- 交易出场后 30 分钟内，价格跌破 P_break × 0.99 且未回到 → **True**
- 否则 → **False**
- AI 可以自动跑这个判定，用户人工复核

## 六、你最容易犯的三个错误（基于 CSV 分析）

### 错误 1：开盘就追

**数据**：你 29% 订单在 09:30-10:00

**对策**：
- **硬规则：09:30-10:00 只观察，一律不下新单**
- 如果手痒，做 Shadow Trade 记录"我当时想追什么"
- 一周后回看，看看如果真追了结果如何 → 大概率会发现 30-40% 是假突破

### 错误 2：同一标的连续下单

**数据**：单日最多 61 笔，个别 symbol 重复出现 6+ 次

**对策**：
- **硬规则：同一 underlying 一天不超过 3 次操作（含开仓、加仓、平仓）**
- 超过第 3 次 → Daily Health Check 标红色警报
- 月度 review 里列出这种"纠缠型"交易的胜率

### 错误 3：深度 OTM / 短 DTE 赌大的

**数据**：最差 5 笔全是 OTM Call，合计亏 $-16K

**对策**：
- **新规：禁止买距离 strike > 5% 的 OTM（尤其 0-1 DTE）**
- Delta 低于 0.25 的期权 **视作"彩票"，仓位不超过 1% 总资金**
- 如果 Regime Score > 80 且 Q3-Q5 全强，才允许 OTM 试仓

## 七、把这份手册内化为肌肉记忆的方法

### 第 1 周

每次你要下单前，在心里默念三个问题：
1. 今天 Regime ≥ 55 吗？
2. Volume 够吗？
3. 能等 Retest 吗？

哪怕你没用系统工具，**光这三个问题的意识就能减少 20% 假突破**。

### 第 2-3 周

开始在 Journal 标注 `entry_method`。每周末花 10 分钟看：
- chase 的胜率是多少？
- retest 的胜率是多少？

数字差异会**震惊你**。然后你自己就愿意多做 retest 了。

### 第 4-6 周

Regime Classifier 上线后：
- 每次下单前看一眼当前 score
- Score < 55 的单自己问："为什么还要做？"

### 3 个月后

你会发现：
- 日均订单从 15 → 8
- 开盘第一小时下单从 29% → 8%
- 胜率从 35% → 45%+
- 账户曲线不再过山车

**这就是从"投机者"到"交易员"的分水岭**。

## 八、应急情境

### 情境 A：你刚追了一个突破，发现是假突破

- **立即止损**，不等"可能会回来"
- 在 Journal 立刻备注："追突破失败，未按手册执行 Q2/Q3/Q4"
- 当天**不要再追任何突破**，改做 retest 或观望
- 周末复盘时单独拿出来看

### 情境 B：你连续 3 次 Retest 入场都失败

- 说明当天 Regime 判断有问题（可能 score 高但大盘后半段转弱）
- **停手，观望 1 小时**
- 重新评估 Regime（有时盘中波动率上升会让原本的判断失效）

### 情境 C：你错过了一个大行情（没追高，结果一路涨走）

- **这是对的决定**。错过一次机会不亏钱；追高一次假突破亏钱
- Journal 备注："遵守手册，错过 X，但无损失"
- 月度 review 时统计"错过的真突破"次数——通常远少于"避开的假突破"次数
